# Software Development Agencies

## Profile
**Category:** Technology
**Market Size:** ~$50B US custom software development services; ~30,000 agencies ranging from 5-person shops to 500+ employee firms
**Tech Maturity:** High (dev tools: GitHub, Jira, Figma, CI/CD) | Low (business operations, profitability tracking, scope management) — agencies build software for others but run their own businesses on spreadsheets and gut feel
**Workforce:** Developers, designers, PMs, QA engineers, tech leads, account managers, agency principals

## Key Pain Themes
Agencies sell time but deliver outcomes, creating a structural tension between fixed-bid pricing and unpredictable scope. A typical agency juggles 5-15 concurrent client projects with shared developer pools, and every week involves triaging which project gets which senior engineer. Scope creep is the default — clients treat "small changes" as free, and project leads lack hard data to push back. Profitability is only knowable in retrospect, often weeks after a project has already gone underwater, because time tracking is inconsistent and the gap between estimated and actual hours is invisible until invoicing.

## Current Tech Landscape
Development tooling is mature (GitHub/GitLab, Jira/Linear, Figma, Vercel/AWS), but these tools serve the build process, not the business process. Project management tools track tasks but not profitability per project or per client. Agencies cobble together Harvest/Toggl for time tracking, spreadsheets for capacity planning, and Slack threads for client communication — none of which connect to produce a real-time view of agency health. PSA tools like Kantata and Teamwork exist but adoption is low among sub-50-person shops.
