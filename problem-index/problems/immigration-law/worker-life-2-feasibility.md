# Feasibility — Incoming Document Classification and File Organization

**Industry:** [[immigration-law|Immigration Law Firms]]
**Problem:** [[problems/immigration-law/worker-life-2|Incoming Document Classification and File Organization]]
**Problem type:** Worker Life

**Problem summary:** Paralegals who receive 20-50 documents per client per case — government notices, passports, diplomas, employment records, tax documents — and manually read, classify, and file each one get an automated intake system that classifies and routes every incoming document without human review except for exceptions.

## Solution Approach
Ship a lightweight assistant that removes repetitive steps for frontline staff, with fast feedback loops and clear handoff to humans. Integrate with case management, add retrieval over statutes/precedents, and ship automation with human oversight.

## Open Source Data
Public statutes and filings are open; matter notes, client data, and outcomes are proprietary, so training/grounding must use firm data.

## Worth Solving?
High for workers — improves retention and quality; commercial ROI is moderate but defensible as part of the platform.
