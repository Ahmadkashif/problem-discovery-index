# Feasibility — Gap Analysis Report Generation from Client Evidence

**Industry:** [[compliance-consulting|Compliance Consulting]]
**Problem:** [[problems/compliance-consulting/low-impact-1|Gap Analysis Report Generation from Client Evidence]]
**Problem type:** Low Impact

**Problem summary:** Generic document generation tools can assemble text but can't structure a compliance gap analysis — mapping each regulatory requirement to client evidence, assigning a compliance status, and generating a risk-scored finding with remediation priority — without deep regulatory domain knowledge.

## Solution Approach
Adapt a horizontal tool with industry-specific templates, rules, and integrations so teams stop working around generic software. Integrate with case management, add retrieval over statutes/precedents, and ship automation with human oversight.

## Open Source Data
Public statutes and filings are open; matter notes, client data, and outcomes are proprietary, so training/grounding must use firm data.

## Worth Solving?
Medium — worthwhile as a customization/efficiency lever once a clear sponsor and data owner are identified.
