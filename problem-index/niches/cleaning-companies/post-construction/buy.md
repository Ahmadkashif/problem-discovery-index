# Project Management Adapted for Phase-Based Cleaning

**Niche:** [[niches/cleaning-companies/post-construction/profile|Post-Construction Cleanup]]
**Industry:** [[industries/cleaning-companies|Cleaning Companies]]
**Type:** Buy & Customize (Vertical Adaptation)
**One-liner:** Generic project management tools don't model the three-phase cleaning workflow (rough, detail, final) or GC milestone dependencies.
**Tags:** #workflow-orchestration #data-integration #automation

## The Problem
Post-construction cleaning happens in three distinct phases: rough clean (remove bulk debris after framing), detail clean (surfaces, fixtures, windows after finishing), and final touch (punch-list items before occupancy). Each phase depends on construction milestones — you can't detail-clean until paint is dry and fixtures are installed. Operators juggle these dependencies across 5-15 active projects using text messages and memory.

## What Already Exists
Asana, Monday.com, and Trello handle generic project management. Construction-side tools like Procore and Buildertrend track GC milestones. Cleaning-specific tools like Swept focus on recurring janitorial. None bridge the gap — connecting GC construction milestones to cleaning phase triggers with crew scheduling and material procurement.

## The Customization Gap
The tool needs construction-phase awareness: when the GC marks "painting complete" in their system, it should trigger the detail-clean phase, notify the crew lead, generate the material order, and schedule the team. It also needs a punch-list workflow — the GC's final walkthrough generates cleaning touch-up items that need to be tracked and closed before CO inspection.

## Target Customer
Post-construction cleaning operators managing 8+ concurrent projects who lose track of phase timing and miss GC-imposed deadlines.

## Impact If Solved
Eliminates missed phase windows (which cause costly re-mobilization) and reduces GC complaints by 40-50%, protecting the operator's referral pipeline.
