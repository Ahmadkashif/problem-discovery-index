# Daily Specials & High Menu Rotation Restaurants

**Parent Industry:** [[industries/independent-restaurants|Independent Restaurants]]
**Category:** Highly Automatable

## Profile
**Market Size:** $20-30B
**Share of Parent Industry:** ~8%
**Digital Adoption:** Low-Medium — POS handles core menu but specials are often rung as "open items" or generic buttons, destroying item-level data
**Target Buyer:** Chef-owner or kitchen manager running daily-changing specials alongside a core menu
**Automation Potential:** High — the daily cycle of cost, price, enter-into-POS, communicate-to-staff is a repetitive, rule-heavy workflow ripe for automation

## What Makes This a Distinct Niche
Restaurants that run daily specials (seafood restaurants with market-priced fish, diners with rotating comfort food, chef-driven concepts with whiteboard menus) face a daily pricing and communication challenge that stable-menu restaurants handle once per quarter. Every morning, the chef decides what to offer based on what was delivered, what needs to be used, and what the market price is — then must calculate a menu price, enter it into the POS, brief the servers, and update the specials board. This daily cycle is almost entirely manual, and the food costing step is usually skipped because there is no time.

## Current Tools & Gaps
POS systems handle the core menu but treat specials as afterthoughts — they must be entered manually each day, often as "open food" items with a typed description and price. This destroys item-level sales data for specials, making it impossible to track which specials sell, which are profitable, and which should be repeated. No tool provides a "daily specials workflow" that combines costing, POS entry, server communication, and menu board updates.
