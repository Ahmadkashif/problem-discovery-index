# Aerial Measurement Integration for Xactimate

**Niche:** [[niches/public-adjusters/xactimate-automation/profile|Xactimate Estimate Automation]]
**Industry:** [[industries/public-adjusters|Public Adjusters]]
**Type:** Buy & Customize (Vertical Adaptation)
**One-liner:** Aerial measurement reports from EagleView and Nearmap provide roof dimensions, but PAs manually re-enter every measurement into Xactimate sketches because the data formats don't directly populate estimate line items.
**Tags:** #automation #data-integration #quick-win

## The Problem
PAs order aerial measurement reports (EagleView, Hover, Nearmap) that provide roof area, pitch, edge lengths, ridge lengths, valley lengths, and face counts. These measurements are exactly what Xactimate needs to calculate line item quantities — but the PA must manually transfer every number from the report into the Xactimate sketch tool, then manually verify that sketch dimensions match the aerial data. This re-entry process takes 45-90 minutes per roof and introduces transcription errors.

## What Already Exists
EagleView provides ESX sketch files that import into Xactimate as a roof diagram, and Xactimate can calculate quantities from sketches. However, the imported sketch often requires manual adjustment (faces that don't align, missing penetrations, incorrect pitch assignments), and the sketch-to-estimate-line-item conversion still requires manual selection of the correct Xactimate codes for each component.

## The Customization Gap
The gap is a seamless pipeline from aerial report to populated estimate: aerial measurements automatically populate not just the Xactimate sketch but also the corresponding line items with correct quantities — shingle squares calculated from roof area, ridge cap linear feet from ridge length, drip edge from eave and rake measurements, steep charges applied based on pitch. The PA receives an estimate with all roof quantities pre-calculated from aerial data, needing only to verify and add non-roof items.

## Target Customer
PAs handling 3+ roof claims per week who order aerial measurement reports but spend 45-90 minutes per claim transferring data into Xactimate.

## Impact If Solved
Eliminates 45-90 minutes of data transfer per roof estimate. Reduces measurement transcription errors that cause carrier estimate disputes. For a PA handling 200 roof claims per year, saves 150-300 hours annually.
