# Prior Authorization Operations

**Parent Industry:** [[industries/medical-supply-retail|Medical Supply Retail]]
**Category:** Highly Automatable

## Profile
**Market Size:** $3-5B (embedded operational cost)
**Share of Parent Industry:** ~7%
**Digital Adoption:** Low-Medium — Brightree provides structured order intake but the actual prior auth process (documentation assembly, payer submission, follow-up) is manual phone/fax/portal work
**Target Buyer:** Billing manager, prior auth specialist, or revenue cycle director at a DME supplier
**Automation Potential:** High — documentation gap analysis, payer-specific requirement matching, and submission routing are all rule-heavy and high-volume processes that consume 40%+ of billing staff time

## What Makes This a Distinct Niche
Prior authorization is the single largest labor cost center in DME operations, consuming 40-50% of billing staff time. Every DME order above a cost threshold requires payer approval before delivery and billing. The process is a matrix of: payer-specific coverage criteria (each payer has different rules), product-specific documentation requirements (wheelchairs require different documentation than CPAP), patient-specific clinical evidence (physician notes, CMN forms, face-to-face records), and submission-specific channel requirements (some payers accept electronic, others require fax, others require portal). First-pass denial rates of 20-35% mean the same work is done 1.5x on average per order. This is the #1 operational pain point in the industry and the highest-ROI automation target.

## Current Tools & Gaps
Brightree provides structured order intake and some payer rule lookups. Parachute Health handles some electronic prescribing for DME. Availity and other clearinghouses handle eligibility verification. No tool provides: pre-submission documentation audit (will this package be approved or denied, and why?), payer-specific gap analysis (what's missing for this specific payer's LCD criteria?), automated submission routing (send to the correct payer channel in the correct format), or denial pattern analysis (which documentation gaps cause the most denials, by payer?).
