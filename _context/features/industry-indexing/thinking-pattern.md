# Industry Indexing — How to Execute

## Step 1: Understand the Workforce

DO this first. ASK: "Who actually does the work day-to-day?"

- IDENTIFY the specific roles (e.g., "Board-certified behavior analysts (BCBAs) supervising registered behavior technicians (RBTs)")
- DO NOT write generic labels like "Healthcare workers"
- IF you don't know the specific job titles → research before writing

## Step 2: Map the Pain Themes

WALK through the workforce's daily operations. ASK for each step:

- "What specific workflow creates friction?"
- "How much time/money does it cost?"
- "What's the consequence when it fails?"

WRITE 3-4 sentences that name actual workflows, not categories.

- GOOD: "RBTs spend 30+ min/session on ABA data collection forms, reducing actual therapy time"
- BAD: "Documentation is a burden"

**IF you catch yourself writing a sentence that could apply to ANY industry → STOP. It's too generic. What's specific to THIS industry?**

## Step 3: Assess Tech Maturity

CALIBRATE to the vertical, not to tech industry standards.

- "High" for a dental practice ≠ "High" for a SaaS company
- ALWAYS include a 1-sentence explanation of what the maturity level means in context
- GOOD: "Medium — EHR adoption is high but interoperability between behavioral health and primary care systems is poor"
- BAD: "Medium"

## Step 4: Map the Tech Landscape

NAME specific tools. EXPLAIN what works and what doesn't.

- GOOD: "Central Reach and CareCloud dominate but lack integrated ABA session data capture"
- BAD: "Several EHR systems exist"

ASK: "What's the gap between what the tool does and what the workflow needs?"

## Step 5: Verify Links

CHECK that the Problems section links to exactly 7 documents using the patterns in `_context/formats/wiki-links.md`. Missing link = broken vault.

## Failure Modes

- **IF your profile reads like a Wikipedia article** → STOP. It's too broad. Add operational specificity — name tasks, time costs, tools.
- **IF pain themes are universal** ("hiring is hard", "costs are rising") → STOP. What's specific to THIS industry?
- **IF tech landscape is just product names** → STOP. Explain the gaps. What does each tool do well, and where does it fail?
- **IF you forgot to link all 7 downstream documents** → STOP. Count them. Must be exactly 7.
